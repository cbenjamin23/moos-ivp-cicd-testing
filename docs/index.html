Choosing A Layer

Use CTest for local C++ contracts. Use a harness when the behavior depends on live MOOS mail, helm state, simulator motion, or ports.

Mission Harness Flow

From scenario to mission result

  1. Stem missions define reusable MOOS community layouts, behavior configs, and baseline geometry.
  2. Harness overlays use small patches so each run isolates one scenario or regression risk.
  3. The harness zlaunch.sh wrapper selects the case and then delegates to the shared xlaunch.sh path, keeping local and CI runs aligned.
  4. pMissionEval reports and shell-side checks reduce the run to compact result lines.
