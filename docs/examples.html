Method

One seemingly benign edit at a time

  1. Start from a passing focused test or harness case.
  2. Make one plausible change without committing it to MOOS-IvP.
  3. Capture the meaningful failure, not merely a compiler error.
  4. Explain the contract that caught the edit and the safer approach.
