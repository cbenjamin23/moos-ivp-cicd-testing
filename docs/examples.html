Method

One change, one explicit contract

  1. State the behavior the change must preserve or introduce.
  2. Run the smallest CTest or mission harness that can prove it.
  3. Capture the meaningful red result, not merely a compiler error.
  4. Show the corrected implementation or real upstream feature commit.
