Behavior Correctness

H01 Trail Behavior

Behavior correctness tests for BHV_Trail. These cases verify relative and absolute trailing geometry, pursuit relevance, runtime trail updates, extrapolation choices, missing-contact behavior, and malformed parameter handling.

Stem Mission

Baseline scenario context

The stem mission uses two simulated vehicles: abe runs BHV_Trail while ben provides the moving contact track. Case overlays vary trailing geometry, relevance settings, runtime updates, and missing-contact behavior.

Examples

Representative runs

Relative Port Trail

Description: Keeps the chaser on the target's port quarter, proving BHV_Trail can hold an offset relative trail angle instead of only following directly astern.

Lead Turn Angle Update

Description: Follows a target through a dogleg while changing trail_angle at runtime, proving the chaser can reframe the desired trail point during a turn.

Runtime Range Extend

Description: Increases trail_range through TRAIL_UPDATES, proving the behavior can accept a runtime range change and settle cleanly at the longer separation.
